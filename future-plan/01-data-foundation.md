# 📊 Data Foundation — Stage 1 & 2 (Market Entry Path)

> **Goal:** Acquire marketable skills in the shortest possible time and start earning.
> **When:** Stage 1 & Stage 2 of the roadmap
> **Why this first:** Becoming an AI Engineer takes 8-12 months. But Excel + SQL + PowerBI can land you work in 2-3 months.

---

<summary>🟢 Stage 1 — Excel + SQL + PowerBI &nbsp;|&nbsp; Market Entry</summary>

<br>

> **Target:** Land your first freelance or local job within 3 months — Data Entry, Reporting, or Dashboard work.

<details>
<summary>Step 1 · Advanced Excel (Week 1–3)</summary>

<br>

> Your C++ logic directly translates here. Excel formulas = programming logic.

**Core Functions**
- [ ] VLOOKUP / XLOOKUP — data matching
- [ ] INDEX + MATCH — flexible lookup (more powerful than VLOOKUP)
- [ ] IF / IFS / SUMIFS / COUNTIFS — conditional logic
- [ ] TEXT, DATE, LEFT, RIGHT, MID — text manipulation
- [ ] UNIQUE, FILTER, SORT — dynamic arrays (Excel 365)

**Data Organization**
- [ ] Pivot Tables — data summarization (the single most important Excel skill)
- [ ] Pivot Charts — visual summaries
- [ ] Data Validation — dropdown lists, input control
- [ ] Conditional Formatting — highlight patterns
- [ ] Named Ranges — clean formula writing

**Power Query (Game Changer)**
- [ ] Import data from CSV, JSON, Web, multiple files
- [ ] Clean messy data — remove duplicates, split columns, unpivot
- [ ] Merge/Append queries — combine multiple data sources
- [ ] Transform steps — repeatable data cleaning pipeline
- [ ] **Learning Power Query puts you ahead of 90% of Excel freelancers**

**Portfolio Project:**
- [ ] **Project 1:** Take a messy sales data file → Clean with Power Query → Summarize with Pivot Table → Create dashboard with charts
- [ ] Add this project to your Upwork portfolio

**Resource:** [Excel Skills for Business (Coursera — Macquarie University)](https://www.coursera.org/specializations/excel) — Free to audit
**Resource:** [Chandoo.org](https://chandoo.org/) — Best free Excel tutorials

</details>

<details>
<summary>Step 2 · SQL Fundamentals (Week 3–6)</summary>

<br>

> SQL is the universal language of data. Data Analyst, BI Analyst, Data Engineer, AI Engineer — everyone needs it. Learn it once, use it forever.

**Basic Queries**
- [ ] SELECT, WHERE, ORDER BY, LIMIT
- [ ] AND, OR, NOT, IN, BETWEEN, LIKE
- [ ] DISTINCT, AS (aliases)
- [ ] NULL handling — IS NULL, COALESCE, IFNULL

**Aggregation (Most Used in Real Jobs)**
- [ ] GROUP BY + Aggregate functions (COUNT, SUM, AVG, MIN, MAX)
- [ ] HAVING — filter after grouping
- [ ] **This is the #1 topic asked in interviews**

**Joins (Critical)**
- [ ] INNER JOIN — matching records
- [ ] LEFT JOIN — all from left + matching from right
- [ ] RIGHT JOIN, FULL OUTER JOIN
- [ ] Self JOIN
- [ ] **Practice:** Write complex joins across 2-3 related tables

**Intermediate**
- [ ] Subqueries — WHERE IN (SELECT ...)
- [ ] CTEs (Common Table Expressions) — WITH clause
- [ ] Window Functions — ROW_NUMBER(), RANK(), LAG(), LEAD(), SUM() OVER()
- [ ] CASE WHEN — conditional logic in SQL
- [ ] String functions — CONCAT, SUBSTRING, TRIM, UPPER, LOWER

**Portfolio Project:**
- [ ] **Project 2:** Create a sample ecommerce database (PostgreSQL) → Answer 10 business questions with SQL → Export results → Visualize in Excel
- [ ] Create a GitHub repo — "SQL Business Analysis Portfolio"

**Setup:** PostgreSQL locally (or use [SQLiteOnline](https://sqliteonline.com/) to start)
**Resource:** [SQLBolt](https://sqlbolt.com/) — Interactive, free, best for beginners
**Resource:** [Mode Analytics SQL Tutorial](https://mode.com/sql-tutorial/) — Real-world focused
**Practice:** [HackerRank SQL](https://www.hackerrank.com/domains/sql) — Solve 2-3 problems daily

</details>

<details>
<summary>Step 3 · PowerBI / Data Visualization (Week 6–10)</summary>

<br>

> Building dashboards = the most visible skill. You can show clients "I built this" — and that sells work.

**PowerBI Basics**
- [ ] Install PowerBI Desktop (Free)
- [ ] Import data — Excel, CSV, PostgreSQL connection
- [ ] Power Query inside PowerBI — data cleaning
- [ ] Data modeling — relationships between tables

**Visualizations**
- [ ] Bar/Column charts, Line charts, Pie charts
- [ ] Cards, KPIs, Gauges — summary metrics
- [ ] Tables and Matrix — detailed views
- [ ] Maps (if location data available)
- [ ] Slicers and Filters — interactive dashboards

**DAX Basics (PowerBI Formula Language)**
- [ ] Measures vs Calculated Columns
- [ ] CALCULATE, FILTER, ALL
- [ ] SUMX, AVERAGEX — iterator functions
- [ ] Time intelligence — SAMEPERIODLASTYEAR, DATEADD
- [ ] **Learning DAX takes your dashboards to a professional level**

**Portfolio Projects:**
- [ ] **Project 3:** Sales Dashboard — monthly trends, top products, regional breakdown
- [ ] **Project 4:** HR Analytics Dashboard — employee turnover, department analysis
- [ ] Record a 2-minute Loom video demo for each project

**Resource:** [Microsoft Learn — PowerBI](https://learn.microsoft.com/en-us/training/powerplatform/power-bi) — Free, official
**Resource:** [Guy in a Cube (YouTube)](https://www.youtube.com/@GuyInACube) — Best PowerBI channel
**Alternative:** Tableau Public (Free) — if you prefer Tableau over PowerBI

<summary>🟡 Stage 2 — Python + Advanced Analytics &nbsp;|&nbsp; Level Up</summary>

<br>

> **Target:** Upgrade from Excel Analyst to Python-powered Data Analyst. Your rate will 2-3x.
> **Why Python:** Excel has limits (1 million rows). Python handles unlimited data. And it opens the door to Backend and AI Engineering.

---

<details>
<summary>Step 5 · Python Fundamentals (Week 1–3)</summary>

<br>

> With your C++ background, Python will feel incredibly easy. The syntax is much simpler.

- [ ] Variables, Data Types (str, int, float, bool, list, dict, tuple, set)
- [ ] Control Flow — if/elif/else, for, while
- [ ] Functions — def, *args, **kwargs, return
- [ ] List Comprehension — `[x for x in range(10) if x > 5]`
- [ ] File I/O — read/write CSV, JSON, text files
- [ ] Error handling — try/except
- [ ] Virtual environments — venv, pip
- [ ] **Mini Project:** CSV file reader — read a sales CSV → calculate totals → print report

**Resource:** [Automate the Boring Stuff with Python](https://automatetheboringstuff.com/) — Free, practical
**Resource:** [Python.org Official Tutorial](https://docs.python.org/3/tutorial/) — Reference

</details>

<details>
<summary>Step 6 · Pandas + Data Manipulation (Week 3–6)</summary>

<br>

> Pandas = Python's Excel. Everything you did in Excel, you'll do in Python — but with 100x more data.

- [ ] DataFrame basics — read_csv, head(), info(), describe()
- [ ] Filtering — `df[df['sales'] > 1000]`
- [ ] Sorting — sort_values()
- [ ] GroupBy + Aggregation — `.groupby('category').sum()` (Python version of Excel Pivot Tables)
- [ ] Merge/Join — `pd.merge()` (Python version of SQL JOIN)
- [ ] Pivot Tables — `df.pivot_table()`
- [ ] Handling missing data — fillna(), dropna()
- [ ] String operations — `.str.contains()`, `.str.split()`
- [ ] Date/Time handling — pd.to_datetime(), .dt accessor
- [ ] **Project 5:** Redo your Excel Project 1 using Python + Pandas — see how much faster it is

**Resource:** [Kaggle — Pandas Course](https://www.kaggle.com/learn/pandas) — Free, hands-on
**Resource:** [Pandas Documentation](https://pandas.pydata.org/docs/) — Reference

</details>

<details>
<summary>Step 7 · Data Visualization with Python (Week 6–8)</summary>

<br>

> PowerBI is great, but Python visualization = more flexible + reproducible in code.

- [ ] Matplotlib basics — line, bar, scatter, pie charts
- [ ] Seaborn — statistical visualizations (heatmaps, distributions, pair plots)
- [ ] Plotly — interactive charts (hover effects, zoom)
- [ ] **Project 6:** Sales Analytics Report — Load data in Jupyter Notebook → Clean → Analyze → Visualize → Write insights in markdown cells

**Resource:** [Kaggle — Data Visualization Course](https://www.kaggle.com/learn/data-visualization) — Free

<details>
<summary>Step 8 · Advanced SQL + Database Skills (Week 8–10)</summary>

<br>

> You learned basic SQL in Stage 1. Now go to the advanced level — this is what bridges you from DA to DE.

- [ ] Advanced Window Functions — PARTITION BY, running totals, moving averages
- [ ] Complex CTEs — recursive CTEs, chained CTEs
- [ ] Subqueries vs Joins — performance comparison
- [ ] EXPLAIN ANALYZE — query performance tuning
- [ ] Indexing strategies — when and why
- [ ] Stored Procedures and Views
- [ ] Database design — normalization (1NF, 2NF, 3NF), star schema
- [ ] **Project 7:** Build a complete PostgreSQL analytics database — design schema → load sample data → write 20 complex queries → create views for dashboards

**Practice:** [DataLemur](https://datalemur.com/) — Real interview SQL questions (Facebook, Amazon, Google)
**Practice:** [StrataScratch](https://www.stratascratch.com/) — Company-specific SQL problems
